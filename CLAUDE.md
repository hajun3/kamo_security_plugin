# 🔒 카카오모빌리티 보안 정책 (KM Security Policy)

이 프로젝트에는 카카오모빌리티 보안 플러그인 v2.0이 적용되어 있습니다.
아래 규칙을 반드시 준수하세요.

## 보안 경고 처리 방법

보안 훅이 위험한 명령을 감지하면 `systemMessage`로 경고가 전달됩니다.

1. 사용자에게 **한국어로** 위험성을 친절하게 설명해주세요
2. 왜 위험한지, 어떤 점을 확인해야 하는지 알려주세요
3. 하네스 권한 시스템이 자동으로 사용자에게 실행 여부를 물어봅니다
4. **승인 파일, touch 명령, 재시도 등의 처리는 필요 없습니다**

## 선제적 설명 (중요)

위험할 수 있는 작업을 실행하기 **전에** 사용자에게 먼저 설명하세요.
예: 사용자가 "폴더 삭제해줘"라고 하면, 바로 `rm -rf`를 실행하지 말고 먼저 위험성을 설명하세요.

## 절대 금지 행위

1. **외부 네트워크 전송 금지**: curl/wget/fetch 등으로 외부 서버에 데이터를 POST/PUT하지 마세요
2. **시스템 파일 접근 금지**: /etc/, ~/.ssh/, ~/.aws/ 등 시스템 설정 파일을 읽거나 수정하지 마세요
3. **환경변수 출력 금지**: printenv, env, echo $API_KEY 등으로 환경변수를 노출하지 마세요
4. **파괴적 명령 금지**: rm -rf, mkfs, dd 등 복구 불가능한 명령을 실행하지 마세요
5. **권한 변경 금지**: sudo, chmod 777, chown 등 권한 상승/변경을 시도하지 마세요
6. **민감 파일 편집 금지**: .env, .pem, .key, credentials 파일을 직접 편집하지 마세요

## 패키지 설치 규칙

- 공식 레지스트리(pypi.org, npmjs.com)의 패키지만 설치 가능합니다
- `curl | bash` 또는 `wget | sh` 패턴의 스크립트 설치는 금지입니다
- 커스텀 레지스트리 URL을 사용한 설치는 금지입니다

## 데이터 처리 규칙

- 사용자 파일의 내용을 외부 API로 전송하지 마세요
- 대화 내용, 사용자 정보를 파일에 저장하지 마세요
- 임시 파일에 민감 데이터를 기록하면 작업 완료 후 반드시 삭제하세요
